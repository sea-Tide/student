# 学生信息管理系统接口文档

## 通用响应

所有接口返回 JSON：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

未登录访问学生接口时由登录拦截器返回 HTTP 401：

```json
{
  "code": 401,
  "message": "未登录！",
  "data": null
}
```

## 登录校验接口

### POST /api/users/login

登录系统，账号密码均来自数据库 `users` 表。

请求体：

```json
{
  "username": "admin",
  "password": "123"
}
```

响应：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {
    "id": 1,
    "username": "admin",
    "displayName": "系统管理员",
    "role": "ADMIN"
  }
}
```

### GET /api/users/me

获取当前登录用户。

### POST /api/users/logout

退出登录并清理服务端 Session。

### POST /api/users/register

注册新用户。注册成功后返回新用户信息，不自动登录。

请求体：

```json
{
  "username": "teacher",
  "displayName": "教师用户",
  "password": "123",
  "confirmPassword": "123"
}
```

响应：

```json
{
  "success": true,
  "message": "操作成功",
  "data": {
    "id": 2,
    "username": "teacher",
    "displayName": "教师用户",
    "role": "USER"
  }
}
```

### POST /api/users/password

修改密码。需要提供账号、原密码和新密码。

请求体：

```json
{
  "username": "teacher",
  "oldPassword": "123",
  "newPassword": "456",
  "confirmPassword": "456"
}
```

响应：

```json
{
  "success": true,
  "message": "操作成功",
  "data": null
}
```

## 学生信息接口

学生接口均要求已登录。

### GET /api/students

查询学生列表，可选参数 `keyword` 支持按学号、姓名、Email、专业模糊查询。

示例：

```http
GET /api/students?keyword=软件
```

### GET /api/students/{id}

按学号查询单个学生。这里的 `id` 就是学号，也是学生表主键。

### POST /api/students

新增学生。

请求体：

```json
{
  "id": "SWEU25004",
  "name": "赵六",
  "email": "zhaoliu@example.com",
  "gender": "男",
  "major": "软件工程",
  "grade": 2026
}
```

### PUT /api/students/{id}

修改学生信息。请求体字段同新增接口，`id` 作为学号主键，修改时不能变更。

### DELETE /api/students/{id}

删除学生。
